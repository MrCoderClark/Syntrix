# 12. User Menu + Search

Avatar dropdown menu with logout + site-wide search.

## Tasks

### A — Avatar dropdown menu (Topbar)

1. **Create `UserMenu` client component** (`frontend/components/shell/UserMenu.tsx`)
   - Fetches `/api/auth/me` (same as Topbar currently does — lift state up or
     keep the fetch here)
   - Renders the Avatar; on click, toggles a dropdown panel
   - Menu items:
     - **My Profile** → `/u/{handle}`
     - **Settings** → `/settings/profile`
     - divider
     - **Sign out** → `POST /api/auth/logout` then redirect to `/signin`
   - Click-outside closes the dropdown
   - `UserMenu.module.css` — absolute-positioned dropdown, shadow, z-index

2. **Add logout endpoint** (`backend/app/auth/router.py`)
   - `POST /api/auth/logout` — clears `access_token` and `refresh_token` cookies
     (set Max-Age=0). No DB work needed (refresh token stays in DB; it'll expire).
   - Returns 200.

3. **Integrate into Topbar** — replace the `<Link><Avatar /></Link>` with
   `<UserMenu />`. Move the `/api/auth/me` fetch into UserMenu (Topbar no
   longer needs it).

4. **Validation:** click avatar → dropdown appears → click "My Profile" →
   navigates to `/u/{handle}`. Click "Sign out" → cookies cleared → redirected
   to `/signin`.

### B — Search

5. **Backend: search endpoint** (`backend/app/search/router.py`)
   - `GET /api/search?q=...&type=all|posts|communities|users&limit=10`
   - Queries:
     - **posts:** `WHERE title ILIKE '%q%' OR body_html ILIKE '%q%'`
       (good enough for Phase 1; `pg_trgm` GIN index is a Phase 2 upgrade)
     - **communities:** `WHERE name ILIKE '%q%' OR slug ILIKE '%q%'`
     - **users:** `WHERE handle ILIKE '%q%' OR display_name ILIKE '%q%'`
   - Response: `{ posts: [...], communities: [...], users: [...] }`
   - Mount on `app.include_router(search_router)`

6. **Frontend: search results page** (`frontend/app/(app)/search/page.tsx`)
   - Client component, reads `?q=` from URL search params
   - Fetches `/api/search?q=...`
   - Three sections: Communities, Users, Posts — each with links
   - Empty state: "No results for '...'"

7. **Wire up Topbar search input**
   - Remove `readOnly` from search input
   - On Enter, navigate to `/search?q={value}`
   - On mobile, show a search icon button that expands or navigates

8. **Validation:** type "homelab" → Enter → results page shows the homelab
   community + any posts containing "homelab".
