# User Popover — Click Any Avatar to Message / View Profile

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a shared `UserPopover` component that appears when clicking any user avatar across the app — showing avatar, name, handle, and actions (Message, View Profile). Replace the current pattern where avatars are inert or only link to profiles.

**Architecture:** One `UserPopover` component (floating card, anchored to click target) used everywhere avatars render. It calls `POST /api/dms/{userId}` on "Message" click, navigates to `/u/{handle}` on "View Profile". The existing `NewDmModal` search remains as a secondary path (the "+" button in the DM sidebar).

**Tech Stack:** React + TypeScript, CSS Modules, existing Avatar component, existing DM API (`POST /api/dms/{userId}`), Next.js `useRouter`.

## Global Constraints

- CSS Modules only (no Tailwind). CSS variables from the design system.
- Fonts: Fraunces (display), Hanken Grotesk (body), JetBrains Mono (code).
- No emoji as icons — use SVG via the `icons.tsx` helper.
- Hand-rolled CSS, no component libraries.
- Popover positioning: CSS-only with anchor logic (no Floating UI lib — YAGNI, we only need bottom-center alignment with viewport clamping).

- [ ] **Step 1: Create the component**

```tsx
// frontend/components/ui/UserPopover.tsx
"use client";

import { useCallback, useEffect, useRef, useState } from "react";
import { useRouter } from "next/navigation";
import styles from "./UserPopover.module.css";

interface UserPopoverProps {
  userId: string;
  handle: string;
  displayName: string;
  avatarUrl?: string;
  triggerRect: DOMRect;
  onClose: () => void;
}

export function UserPopover({
  userId,
  handle,
  displayName,
  avatarUrl,
  triggerRect,
  onClose,
}: UserPopoverProps) {
  const router = useRouter();
  const ref = useRef<HTMLDivElement>(null);
  const [messaging, setMessaging] = useState(false);

  const initials = displayName
    .split(" ")
    .map((w) => w[0])
    .join("")
    .slice(0, 2)
    .toUpperCase();

  const handleMessage = useCallback(async () => {
    setMessaging(true);
    try {
      const res = await fetch(`/api/dms/${userId}`, { method: "POST" });
      if (res.ok) {
        const room = await res.json();
        router.push(`/chat?room=${room.id}`);
      }
    } finally {
      setMessaging(false);
    }
    onClose();
  }, [userId, router, onClose]);

  const handleViewProfile = useCallback(() => {
    router.push(`/u/${handle}`);
    onClose();
  }, [handle, router, onClose]);

  // Close on outside click or Escape
  useEffect(() => {
    function handleClick(e: MouseEvent) {
      if (ref.current && !ref.current.contains(e.target as Node)) onClose();
    }
    function handleKey(e: KeyboardEvent) {
      if (e.key === "Escape") onClose();
    }
    document.addEventListener("mousedown", handleClick);
    document.addEventListener("keydown", handleKey);
    return () => {
      document.removeEventListener("mousedown", handleClick);
      document.removeEventListener("keydown", handleKey);
    };
  }, [onClose]);

  // Position below trigger, centered
  const top = triggerRect.bottom + 8;
  const left = triggerRect.left + triggerRect.width / 2;

  return (
    <div
      ref={ref}
      className={styles.popover}
      style={{ top, left }}
      role="dialog"
      aria-label={`${displayName}'s profile`}
    >
      <div className={styles.avatarWrap}>
        {avatarUrl ? (
          <img src={avatarUrl} alt="" className={styles.avatarImg} />
        ) : (
          <span className={styles.avatarFallback}>{initials}</span>
        )}
      </div>
      <div className={styles.name}>{displayName}</div>
      <div className={styles.handle}>@{handle}</div>
      <div className={styles.divider} />
      <button
        className={styles.action}
        onClick={handleMessage}
        disabled={messaging}
      >
        {messaging ? "Opening..." : "Message"}
      </button>
      <button className={styles.action} onClick={handleViewProfile}>
        View Profile
      </button>
    </div>
  );
}
```

### Task 2: Create ClickableAvatar Wrapper

Instead of modifying every call site's Avatar directly, create a thin wrapper that handles click → popover state. This way each integration site just swaps `<Avatar>` for `<ClickableAvatar>` with a few extra props.

**Files:**
- Create: `frontend/components/ui/ClickableAvatar.tsx`
- Create: `frontend/components/ui/ClickableAvatar.module.css`

**Interfaces:**
- Consumes: `Avatar` component, `UserPopover` component
- Produces: `<ClickableAvatar userId handle displayName avatarUrl size />` — renders Avatar, opens UserPopover on click

- [ ] **Step 1: Create the wrapper component**

```tsx
// frontend/components/ui/ClickableAvatar.tsx
"use client";

import { useCallback, useRef, useState } from "react";
import { createPortal } from "react-dom";
import { Avatar } from "./Avatar";
import { UserPopover } from "./UserPopover";
import styles from "./ClickableAvatar.module.css";

interface ClickableAvatarProps {
  userId: string;
  handle: string;
  displayName: string;
  avatarUrl?: string;
  size?: "xs" | "sm" | "md" | "lg" | "xl";
}

export function ClickableAvatar({
  userId,
  handle,
  displayName,
  avatarUrl,
  size = "sm",
}: ClickableAvatarProps) {
  const [open, setOpen] = useState(false);
  const [rect, setRect] = useState<DOMRect | null>(null);
  const btnRef = useRef<HTMLButtonElement>(null);

  const initials = displayName
    .split(" ")
    .map((w) => w[0])
    .join("")
    .slice(0, 2)
    .toUpperCase();

  const handleClick = useCallback((e: React.MouseEvent) => {
    e.preventDefault();
    e.stopPropagation();
    if (btnRef.current) setRect(btnRef.current.getBoundingClientRect());
    setOpen(true);
  }, []);

  return (
    <>
      <button
        ref={btnRef}
        className={styles.trigger}
        onClick={handleClick}
        aria-label={`${displayName}'s actions`}
        type="button"
      >
        <Avatar
          src={avatarUrl}
          alt={displayName}
          fallback={initials}
          size={size}
        />
      </button>
      {open &&
        rect &&
        createPortal(
          <UserPopover
            userId={userId}
            handle={handle}
            displayName={displayName}
            avatarUrl={avatarUrl}
            triggerRect={rect}
            onClose={() => setOpen(false)}
          />,
          document.body,
        )}
    </>
  );
}
```

### Task 7: Viewport Edge Clamping

**Files:**
- Modify: `frontend/components/ui/UserPopover.tsx`
- Modify: `frontend/components/ui/UserPopover.module.css`

**Interfaces:**
- Consumes: `triggerRect` from ClickableAvatar
- Produces: Popover that doesn't overflow the viewport

When the avatar is near the bottom or right edge of the viewport, the popover should flip above or shift horizontally. Simple JS clamping in the component — no library needed.

- [ ] **Step 1: Add viewport clamping logic**

In `UserPopover.tsx`, after calculating `top` and `left`, clamp:

```tsx
const POPOVER_WIDTH = 240;
const POPOVER_HEIGHT_ESTIMATE = 200;
const MARGIN = 8;

let top = triggerRect.bottom + MARGIN;
let left = triggerRect.left + triggerRect.width / 2;

// Flip above if too close to bottom
if (top + POPOVER_HEIGHT_ESTIMATE > window.innerHeight - MARGIN) {
  top = triggerRect.top - POPOVER_HEIGHT_ESTIMATE - MARGIN;
}

// Clamp horizontal
const halfWidth = POPOVER_WIDTH / 2;
if (left - halfWidth < MARGIN) left = halfWidth + MARGIN;
if (left + halfWidth > window.innerWidth - MARGIN) left = window.innerWidth - halfWidth - MARGIN;
```
