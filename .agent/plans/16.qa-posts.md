# Section 16 — Q&A Post Type

Branch: `feat/qa-posts`

## What ships

- `post_type` column on posts (`'discussion'` | `'question'`)
- `answer_count` and `has_accepted_answer` denormalized columns on posts
- `answers` table with voting, mod-remove, soft-delete
- `answer_votes` table
- Triggers: answer score maintenance, answer_count on posts
- Backend: full answer CRUD + answer voting endpoints
- Frontend: "Ask a question" flow (post type selector on submit page)
- Frontend: question detail page with answer list, answer editor, voting
- Frontend: tag picker on question submit (reuses existing TagPicker)
- Feed filtering: community feed can filter by post_type
- Question cards in feeds show answer count + accepted badge

### T1 — Migration: add post_type + answer tables

**File:** `backend/alembic/versions/<auto>_add_qa_post_type_and_answers.py`

Schema changes:

```sql
-- Extend posts
ALTER TABLE syntrix.posts
  ADD COLUMN post_type text NOT NULL DEFAULT 'discussion'
    CHECK (post_type IN ('discussion', 'question')),
  ADD COLUMN answer_count integer NOT NULL DEFAULT 0,
  ADD COLUMN has_accepted_answer boolean NOT NULL DEFAULT false;

-- Answers
CREATE TABLE syntrix.answers (
  id            uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  question_id   uuid NOT NULL REFERENCES syntrix.posts(id) ON DELETE CASCADE,
  author_id     uuid REFERENCES syntrix.users(id) ON DELETE SET NULL,
  body_json     jsonb NOT NULL,
  body_html     text NOT NULL,
  score         integer NOT NULL DEFAULT 0,
  is_accepted   boolean NOT NULL DEFAULT false,
  accepted_at   timestamptz,
  deleted_at    timestamptz,
  removed_at    timestamptz,
  removed_by    uuid REFERENCES syntrix.users(id) ON DELETE SET NULL,
  removed_reason text,
  created_at    timestamptz NOT NULL DEFAULT now(),
  updated_at    timestamptz NOT NULL DEFAULT now()
);

-- Answer votes
CREATE TABLE syntrix.answer_votes (
  id         uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  answer_id  uuid NOT NULL REFERENCES syntrix.answers(id) ON DELETE CASCADE,
  user_id    uuid NOT NULL REFERENCES syntrix.users(id) ON DELETE CASCADE,
  value      smallint NOT NULL CHECK (value IN (-1, 1)),
  created_at timestamptz NOT NULL DEFAULT now(),
  UNIQUE(answer_id, user_id)
);

-- Triggers
-- 1. answer_votes → answers.score (same pattern as post_votes → posts.score)
-- 2. answers INSERT/DELETE → posts.answer_count
```

**Validation:** `make db-migrate` succeeds, `\d syntrix.answers` shows correct columns.

### T4 — Answer router (CRUD + voting)

**File:** `backend/app/answers/router.py` (new)

Endpoints:

| Method | Path | Description | Auth |
|--------|------|-------------|------|
| POST | `/api/posts/{post_id}/answers` | Post an answer | member |
| GET | `/api/posts/{post_id}/answers` | List answers for a question | public |
| PATCH | `/api/answers/{answer_id}` | Edit own answer | author |
| DELETE | `/api/answers/{answer_id}` | Soft-delete own answer | author/admin |
| POST | `/api/answers/{answer_id}/remove` | Mod-remove an answer | mod/owner/admin |
| POST | `/api/answers/{answer_id}/accept` | Accept answer (question author only) | question author |
| DELETE | `/api/answers/{answer_id}/accept` | Unaccept answer | question author |
| POST | `/api/answers/{answer_id}/vote` | Vote on answer | member |
| DELETE | `/api/answers/{answer_id}/vote` | Remove vote | member |

**Authorization rules:**
- Only members of the community can post answers
- Only the answer author can edit
- Only the question author can accept/unaccept
- Accept sets `is_accepted=true`, `accepted_at=now()` on the answer AND
  `has_accepted_answer=true` on the post. Unaccept reverses both.
- Only one answer can be accepted per question — accepting a new one
  unaccepts the previous.
- Voting follows same pattern as PostVote: value ∈ {-1, 1}, toggle removes

**Answer listing sort order:** accepted answer first, then by score desc,
then by created_at asc (oldest first among same score).

**Validation:** API tests for each endpoint (happy path + auth failures).
