# 15. Tags

Community-scoped tags for categorizing questions. Foundation for the Q&A
layer — questions will require at least one tag.

## Tasks

### A — Backend: Tag model + CRUD

1. **Create Tag model** in `backend/app/models/tag.py`:
   - `id` (uuid PK), `community_id` (FK → communities), `name` (citext),
     `slug` (citext), `description` (text, nullable), `color` (text, nullable),
     `usage_count` (int, default 0), `created_by` (FK → users),
     `created_at` (timestamptz)
   - Unique constraint on `(community_id, slug)`

2. **Create QuestionTag join model** in same file:
   - `question_id` (FK → posts), `tag_id` (FK → tags)
   - Composite PK `(question_id, tag_id)`

3. **Apply DDL** via `docker exec` (consistent with Phase 1 approach).

4. **Register models** in `backend/app/models/__init__.py`.

5. **Create tag schemas** in `backend/app/tags/schemas.py`:
   - `TagResponse(id, slug, name, description, color, usage_count, community_slug)`
   - `TagCreateRequest(name, description?, color?)` — auto-generates slug
   - `TagUpdateRequest(name?, description?, color?)`

6. **Create tag router** at `backend/app/tags/router.py`:
   - `GET /api/communities/{slug}/tags` — list all tags for a community
   - `GET /api/communities/{slug}/tags/{tag_slug}` — single tag + usage count
   - `POST /api/communities/{slug}/tags` — create tag (mod/owner/admin only)
   - `PATCH /api/communities/{slug}/tags/{tag_slug}` — update (mod/owner/admin)
   - `DELETE /api/communities/{slug}/tags/{tag_slug}` — delete (admin only,
     only if usage_count == 0)
   - `GET /api/communities/{slug}/tags/autocomplete?q=` — search tags by
     prefix (ILIKE), limit 10, for the tag picker

7. **Register router** in `backend/app/main.py`.

8. **Validation:** `curl` test all endpoints — create, list, autocomplete,
   update, delete.

### B — Frontend: Tag management UI

9. **Tag pills component** — `frontend/components/ui/TagPill.tsx`:
   - Small pill showing tag name with optional color dot
   - Used in tag lists, question cards, tag picker results

10. **Tag list on community page** — show tags below the community header
    on `/c/{slug}`. Clicking a tag filters to questions with that tag
    (deferred to Section 15 when questions exist — for now, just show the
    tags as a row of pills).

11. **Tag management in community settings** (admin/mod only):
    - Inline form to create a new tag (name, description, color picker)
    - List existing tags with edit/delete buttons
    - Route: part of `/c/{slug}/settings` or a dedicated
      `/c/{slug}/settings/tags` page

12. **Tag autocomplete component** — `frontend/components/ui/TagPicker.tsx`:
    - Input that queries `/api/communities/{slug}/tags/autocomplete?q=`
    - Dropdown showing matching tags
    - Click to select, shows as pills below the input
    - Max 5 tags per question
    - Will be used by the "Ask Question" form in Section 15

13. **Validation:** visit a community → see tags row. As admin, create/edit/
    delete tags. Use the autocomplete picker — results appear as you type.
